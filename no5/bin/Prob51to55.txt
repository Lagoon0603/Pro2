Prob52
実行結果：
1)A:m1 A:m2 A:m3 
2)B:m1 A:m2 A:m3 
3)C:m1 C:m2 A:m3 
4)C:m1 C:m2 A:m3 

理由：
・A v1 = new A();の場合
インスタンスはAクラスである。
クラスAではm1(), m2(), m3()が定義されているため、すべてAクラスのメソッドがそのまま呼ばれ、A:m1 A:m2 A:m3 と出力される。

・B v2 = new B(); の場合
インスタンスはBクラス（Aのサブクラス）である。
m1(): Bクラス内でオーバーライドされているため、B:m1が出力される。
m2(): Bクラス内でsuper.m2()を呼び出しているため、親クラスAのm2()が実行され A:m2が出力される。
m3(): Bクラス内には定義されておらず、親クラスAから継承しているため、A:m3が出力される。

・C v3 = new C();の場合
インスタンスはCクラス（Bのサブクラス）である。
m1(): Cクラス内でオーバーライドされているため、C:m1が出力される。
m2(): Cクラス内でオーバーライドされているため、C:m2が出力される。
m3(): Cクラス内でsuper.m3()を呼び出している。
親クラスのBにはm3()がないため、さらにその親のAクラスのm3()が呼び出され、A:m3が出力される。

・User v4 = new User();の場合
インスタンスはUserクラス（Cのサブクラス）である。
Userクラス自身はm1(), m2(), m3()を１つもオーバーライドしていない。
そのため、直近の親クラスであるCクラスのメソッドがそのまま引き継がれる。
結果として、C v3 = new C();と全く同じ挙動となり、C:m1 C:m2 A:m3が出力される。


Prob53
コンパイルエラーとなる箇所は (1)、(5)、(7)

(1) d = 1;
変数dはクラスDの参照型である。
そこにint型の数値1を直接代入しようとしているため、型の不適合によりエラーとなる。

(5) x = 5;
xはスーパークラスD内でprivateと宣言されている。
private変数はサブクラスであるEには継承されないため、Eクラス内からxに直接アクセスを行うことはできないから。

(7) e.x = 7;
変数eの型であるEクラスには、スーパークラスDからprivate変数xが継承されていない。
したがって、Eのインスタンスに対してe.xという形で対象の変数を見つけることができず、エラーになる。
