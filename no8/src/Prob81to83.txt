理由
methodA("Java") 呼び出し時、tryブロック内が正常に終了しても finally ブロックは必ず実行されるため "The finally program is Java" と出力される。
その後 "Program = Java" を出力する。
次に methodA("Smalltalk") を呼び出すと Exception が throw されるが、メソッドを抜ける前に必ず finally ブロックが実行されるため "The finally program is Smalltalk" が出力される。
その後、例外は呼び出し元のmainメソッドに伝播し、catch ブロックで "I don't know Smalltalk!" が出力され、最後に "The actual end" が出力される。

Throw4.java
実行結果
Before a
Before b
Before c
c: finally
b: finally
a: java.lang.NumberFormatException: For input string: "6.12"
a: finally
After a
main: finally

理由
メソッドc()で "6.12" をIntegerに変換しようとして NumberFormatException が発生し throw される。
c()のfinallyブロックが実行され "c: finally" を出力して例外がメソッドb()へ伝播する。
b()では ArithmeticException しか catch しないため、例外は処理されず b()のfinallyブロック "b: finally" が実行された後、メソッドa()へ伝播する。
a()では NumberFormatException を catch するため、"a: " とともに例外の詳細メッセージが出力される。
その後 a()のfinallyブロック "a: finally" が実行され、a()で例外が処理されたため、呼び出し元のmain()は正常に処理を継続し "After a" を出力する。
最後にmain()のfinallyブロック "main: finally" が実行される。

Throw5.java
理由
catch ブロックは記述された順に上から評価されるが、最初の catch ブロックで指定されている Exception クラスは、次の catch ブロックで指定されている ArithmeticException クラスのスーパークラスである。
そのため、ArithmeticException が発生しても最初の Exception の catch ブロックで捕捉されてしまい、2つ目の catch ブロックには絶対に到達できない。
Javaでは到達不能コードが存在する場合、コンパイルエラーとなるからである。
